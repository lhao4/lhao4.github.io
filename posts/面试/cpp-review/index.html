2、函数重载问题#

什么是函数重载？#

在同一个作用域内，函数名字相同，参数的个数、顺序或类型其中一个不同就是函数重载，函数重载与返回值无关。

C++为什么能函数重载？#

因为C++编译器在编译函数的时候会在原来名字的基础上加上参数类型来识别不同的重载函数，但是C语言函数名被定义成什么编译后任然是原名，所以C语言不能重载C++可以重载。

重载和参数默认值的冲突（一般不会问，可以主动说，也可以说重载是静态多态）#

using namespace std;

int main(){
    fun(0);//此时编译器不知道调用那个函数
}

3、引用#

特点：#

  • 必须初始化，且不能初始化为空
  • 不能修改引用关系
  • 因为引用的底层时指针常量（int * const p）指针不能改变指向。

引用类型：#

  • 左值引用：只能引用左值（有名字的能找到地址的是左值）
  • 右值引用：只能引用右值（每日名字的找不到地址的，例如函数的返回值，这里可以引申到移动语义，例如：移动复制 、移动构造等）
  • 万能引用：既可以引用左值，也可以引用右值

引用的用处：#

  • 用在函数的参数，可以替换指针，使代码变得更简洁
  • 用在函数的返回值，函数的返回值本身使右值，但是如果想让函数的返回值为左值，那么可以返回引用类型。但是需要注意的是如下代码：
  • 还可以避免拷贝构造

（重点）引用和指针的区别：#

  • 引用不可以为空，但是指针可以为空
  • 指针是一个实体，需要分配内存空间；引用只是变量的别名，不需要分配内存空间
  • 引用在定义是必须初始化，指针则没有要求、
  • 引用不可以改变指向，但是指针可以改变指向
  • 引用比指针使用起来形式上更方便简洁
  • 引用比指针更加安全（指针存在野指针、访问空指针，悬挂指针等）
  • 引用不能多级，但是指针可以有多级。（引用有万能引用、左值引用、右值引用可以说，然后引导说移动语义）
  • 访问实体的方式不同，指针需要显示解引用，引用编译器自己处理
  • sizeof引用得到的是所指向的变量（对象）的大小，而sizeof指针得到的是指针变量本身的大小
  • 指针作为函数参数传递时传递的是指针变量的值，而引用作为函数参数传递时传递的时实参本身
  • 自增运算结果不同
//指针变量自增运算
int a[5] = { 1,2,3,4,5 };
int* p = a;
cout << *p << endl;  // 输出1，而不是注释中的10
cout << *(p++) << endl;  //1，先输出1，然后指向2
cout << *p << endl;
//引用自增运算
int& r = a[0];
cout << r << endl;  // 输出1
cout << (r++) << endl;  // 输出1，之后a[0]的值变为2
cout << r << endl;
//总结：指针的自增是改变指针指向的地址，而引用的自增是修改它所引用的变量的值

4、（重点）面向对象的三大特征#

面向对象：把数据和对数据的操作方法放在一起，构成一个相互依靠的整体，称为对象，对同类对象抽象出共同特性，类中大多数数据只能用本类的方法进行处理。

  • 封装：将事物的属性（成员变量）和行为（成员函数）封装在一起形成一个类。并且可以设置相应的访问权限（私有、守保护的、公有）。
  • 继承：可以通过现有的类来实现新的类，相当于对现有的类进行扩展，提高了代码的复用性，可维护性，是多态的前提（多继承问题、菱形继承问题）。
  • 多态：父类的指针或者引用指向子类对象，开启多态的前提条件是调用重写的函数。可以让同一个行为在不同的对象上有不用的表现。

5、new和malloc的区别#

  • new的返回值不需要强制转换，malloc的返回值需要强制转换
  • new是运算符可以重载（重载是静态多态，可以引导说多态），malloc是库函数（malloc不能重载因为他是C语言的库函数，可以说C和C++的编译方式的区别）
  • new不需要传入想要申请的具体字节个数，malloc需要传入想要申请的具体的字节个数
  • malloc申请失败例如传入参数是负数的情况下返回空，new申请失败会抛出异常（引出异常机制，以及如何实现自定义异常）
  • 如果给一个类分配堆区内存new会先调用malloc分配内存，然后再调用构造函数给成员变量赋值（注意：构造函数给成员变量赋值可以引出初始化参数列表）
  • new申请的堆区内存需要使用delete释放，delete先调用析构函数再调用free（free也是C语言的库函数和malloc是配套的）这里可以说一下为什么先调用析构再调用free
#include<iostream>
using namespace std;

int main() {
    //malloc
	int* p = (int*)malloc(8);//在堆区开辟8个字节的空间

	//malloc失败
	int* p1 = (int*)malloc(-6);
	if (p1 == NULL) cout << "malloc失败" << endl;
	
	//new
	int* t = new int(3);//在堆区开辟4个字节的空间，并初始化为3
	cout << "*t = " << *t << endl;

	int* t1 = new int[3];//在堆区开辟3个int类型的数组空间
    return 0;
}

6、构造函数#

  • 创建对象的时候编译器自动调用构造函数
  • 构造函数和类名相同，没有返回值也不写void
  • 如果没有实现构造函数编译器会提供默认空实现的无参构造函数
  • 构造函数时给成员变量赋值的不是初始化（引出初始化参数列表）
  • 有兴趣的同学可以研究一下（移动构造，继承构造，委托构造）
#include<iostream>
#include<type_traits>
using namespace std;

class Base {
public:
    // 1. 默认构造函数
    Base() {
        a = nullptr;
        cout << "Base默认构造函数被调用" << endl;
    }

    // 2. 有参构造函数
    Base(int n) {
        a = new int(n);
        cout << "Base有参构造函数被调用，值为：" << n << endl;
    }

    // 3. 拷贝构造函数
    Base(const Base& other) {
        if (other.a != nullptr) {
            a = new int(*other.a);
            cout << "Base拷贝构造函数被调用，拷贝值为：" << *a << endl;
        }
        else {
            a = nullptr;
            cout << "Base拷贝构造函数被调用，拷贝空对象" << endl;
        }
    }

    // 4. 移动构造函数
    Base(Base&& other) noexcept {
        a = other.a;
        other.a = nullptr; // 确保other.a被设置为nullptr
        cout << "Base移动构造函数被调用" << endl;
    }

    // 5. 转换构造函数
    Base(double d) {
        a = new int(static_cast<int>(d));
        cout << "Base转换构造函数被调用，转换值为：" << *a << endl;
    }

    // 6. 委托构造函数
    Base(int n, bool flag) : Base(n) {
        if (flag) {
            *a *= 2;
            cout << "Base委托构造函数额外处理，值变为：" << *a << endl;
        }
    }

    ~Base() {
        if (a != nullptr) {
            delete a;
            a = nullptr;
        }
        cout << "Base析构函数被调用" << endl;
    }

private:
    int* a = nullptr;
    const int a_val = 0;
};

class A : public Base {
public:
    A() : Base(), b(nullptr) {} // 初始化b为nullptr

    // 使用继承构造函数，但需要处理b的初始化
    using Base::Base;

    // 修复继承构造函数不初始化b的问题：提供额外的构造函数
    A(int n) : Base(n), b(new int(n)) {
        cout << "A的int构造函数被调用，b的值为：" << *b << endl;
    }

    // 9. 显式构造函数
    explicit A(long l) : Base(static_cast<int>(l)), b(new int(static_cast<int>(l))) {
        cout << "A显式构造函数被调用，值为：" << *b << endl;
    }

    // 10. 模板构造函数
    template <typename T>
    A(T t) : Base(static_cast<int>(t)), b(new int(static_cast<int>(t))) {
        static_assert(!std::is_same_v<T, int> && !std::is_same_v<T, long>,
            "int/long类型请使用父类继承的构造函数");
        cout << "A模板构造函数被调用，值为：" << *b << endl;
    }

    // 拷贝构造函数
    A(const A& other) : Base(other) {
        if (other.b != nullptr) {
            b = new int(*other.b);
        }
        else {
            b = nullptr;
        }
        cout << "A拷贝构造函数被调用" << endl;
    }

    ~A() {
        if (b != nullptr) {
            delete b;
            b = nullptr;
        }
        cout << "A析构函数被调用" << endl;
    }

private:
    int* b = nullptr;
};

int main() {
    Base b1;
    Base b2(10);
    Base b3(b2);
    Base b4(std::move(b3));
    Base b5 = 3.14;
    Base b6(5, true);

    cout << "---------------------------------------------" << endl;

    A a1;
    A a2(20);
    A a3 = static_cast<A>(1000000000L);
    A a4('B');
    A a5(a4);

    return 0;
}

7、析构函数#

  • 对象被释放时编译器会自动调用析构函数
  • 析构函数的函数名为~类名，没有返回值也不写void，也没有参数
  • 如果没有实现析构函数编译器会提供默认的析构函数
  • 析构函数是释放对象成员变量指向的堆区内存的
class A {
    int* p = nullptr;
public:
    A(int n) {//实现了构造函数编译器不在提供默认的无参构造
        if (n > 0) {
			p = new int[n];//在堆区分配n个int类型的空间
        }
    }
    ~A() {
        if (p) {
			delete[] p;//注意释放数组时要加[]，否则只会释放第一个元素，导致内存泄漏
        }
    }
};

8、拷贝构造#

拷贝构造的调用时机：#

  • 用已经存在的对象初始化新的对象时
  • 当对象以值的形式作为函数的参数或返回值时

拷贝构造的参数不是引用可以吗？#

不行，拷贝构造的参数如果不是引用会导致无限递归，前面加上const是为了避免通过形参修改实参。

深拷贝和浅拷贝有什么区别：#

  • 深拷贝：拷贝相同大小的内存并且数值也相同
  • 浅拷贝：简单的赋值操作，编译器提供的拷贝构造是浅拷贝

9、const#

  • const修饰变量时表示变量不可以修改，但是可以通过指针来修改
const int a = 10;
const int* p = &a;
//首先将指针p强制转换为int*类型，然后通过该指针修改a的值
int* p1 = const_cast<int*>(p);
*p1 = 20; // 修改a的值
cout << a << " " << *p << " " << *p1 << endl;
  • const修饰指针时
const int* p;//常量指针，p指向的内容不可变
//int const* p;
//*p = 20; //错误，不能通过p修改a的值

int a = 10;
int* const p2 = &a; //指针常量，p2的指向不可变，并且必须初始化
*p2 = 20; //正确，可以通过p2修改a的值
int b = 30;
//p2 = &b; //错误，不能修改p2的指向
//总结：const修饰指针时，靠近*号的一方不可变
  • const修饰函数时，表示常函数
    • 常函数内的this指针在vs中是const type *this;(type代表类型)，所以常函数内不能修改非mutable的成员变量，也不能调用非常函数(因为this指针在传递的时候类型不匹配)
    • 常对象只能调用常函数，非常对象既可以调用常函数也可以调用非常函数
    • 常函数和非常函数是重载关系因为this类型不同（可以引导说this是怎么来的）
class MyClass {
public:
    // 非常函数：this 类型为 MyClass* const
    void print() {
        cout << "非常函数：可修改成员" << endl;
    }

    // 常函数：this 类型为 const MyClass* const
    void print() const {  // 与上面的 print 构成重载
        cout << "常函数：不可修改成员" << endl;
    }
};

int main() {
    MyClass obj;         // 非常对象
    const MyClass cobj;  // 常对象

    obj.print();   // 调用非常函数（匹配 MyClass* const 类型的 this）
    cobj.print();  // 调用常函数（匹配 const MyClass* const 类型的 this）
    return 0;
}

10、static#

static修饰全局变量时：#

如果没有初始化，编译器会自动初始化为0，生存周期和全局变量和程序一致，只能在当前文件访问，如果想在头文件中定义全局变量，那么需要定义成静态全局变量，因为如果多个cpp文件引入同一个头文件，那么会导致多个cpp文件中有相同的全局变量，如果没有定义为static会导致重定义问题。

static修饰局部变量时：#

只能用在局部作用域，只能被初始化一次，不会随着函数的结束而被释放，观察下面代码结构即可得知：

void fun(){
    static int a = 1;
    a++;
    cout<<a<<endl;
}
int main(){
    fun();
    fun();
}

static修饰成员变量时：#

  • 静态成员变量在主函数之前被初始化也就是编译期间初始化
  • 公有的静态成员变量可以通过类名 :: 和 对象名直接访问
  • 静态成员变量不会占有对象的内存，单独的存放在静态区
  • 静态成员变量必须在类外初始化
  • 静态成员也可以实现单例模式

static修饰成员函数时：#

静态成员函数没有this指针，因此静态成员函数不能访问非静态成员变量(原因：无法区分是哪个对象的成员变量)。

静态成员函数不能调用非静态成员函数，因为非静态成员函数有隐含参数this，因为静态成员函数无法给非静态成员函数的this传值。

静态成员函数可以调用静态成员变量因为静态成员变量是共享的不需要使用this来区分是哪个对象的，也可以调用静态成员函数，因为静态成员函数不需要给this传值

11、函数重写（函数覆盖）& 隐藏#

重写：#

在继承关系中父类和子类的函数：返回值、名字、参数都相同并且父类中的函数必须是虚函数这就是函数重写。

class father {
public:
	virtual void fun() {
		cout << "father" << endl;
	}
};
class son :public father {
public:
	virtual  void fun() {//子类函数的virtual写不写都可以
		cout << "son" << endl;
	}
};

隐藏：#

在继承关系中父类和子类的函数：名字相同不是重写就是隐藏。发生函数隐藏时，函数被调用的时候看调用该函数的指针或者对象是什么类型的，就调用哪个类中的隐藏函数。

注：这里一般会问函数重载、隐藏、重写的区别，根据定义来说就可以

class father {
public:
	virtual void fun() {

12、多态#

多态分为静态多态和动态多态，静态多态是在编译期间就能确定的多态，动态多态是在运行期间确定的多态。

静态多态包括：函数重载、运算符重载、模板。

动态多态：父类指针或者引用指向子类对象，并通过指针或引用调重写函数。

动态多态的调用过程#

首先会通过父类指针或者引用访问到子类对象中的虚表指针，然后通过虚表指针找到虚函数表，通过虚函数表存放的虚函数地址去调用虚函数。

虚函数表：#

在编译阶段被构造，每个类只有一个虚函数表，所有对象共享同一个虚函数表，不是每个对象拥有一个。

虚函数表指针：#

当类中存在虚函数时，编译器会给类增加一个指针类型的成员变量，增加的这个指针变量就是虚函数表指针，创建对象虚表指针就会被创建，并且在构造函数中被赋值，每个对象都有自己的虚表指针。

13、纯虚函数&抽象类#

在虚函数的基础上函数体的部分被 = 0 就是纯虚函数。纯虚函数在虚函数表中存放的是0，虚函数则存放的是虚函数的地址，拥有纯属函数的类则被称为纯虚类（抽象类），纯虚类不能实例化对象，一般来说只作为基类，子类如果不重写父类中的纯虚函数子类也是抽象类。

15、堆和栈#

内存的角度：#

堆需要手动申请手动释放内存，在C++中一般使用new，delete，C语言中使用malloc，free（引出new和malloc的区别），栈区对象不需要手动释放，超出作用域编译器会自动回收，由于栈是操作系统来管理的所以栈的申请释放速度要比堆区的快（了解一下用户态和内核态）

补充：用户态与内核态（解释 “堆速度慢” 的底层原因）#

程序运行时存在两种特权级别，内存分配的 “速度差异” 本质是是否需要切换特权级：

  • 用户态：程序正常运行的普通权限级别，只能访问自身地址空间，无法直接操作硬件或系统资源（如内存分配）。
  • 内核态：操作系统内核运行的高权限级别，可直接管理内存、硬件等核心资源。

当使用 <font style="color:rgba(0, 0, 0, 0.85) !important;">malloc</font> 或 <font style="color:rgba(0, 0, 0, 0.85) !important;">new</font> 申请堆内存时：

  1. 程序先在用户态检查自身维护的 “内存缓存池”（若有空闲则直接分配，避免内核态切换）；
  2. 若缓存池无可用内存，则需触发系统调用（如 Linux 的 <font style="color:rgb(0, 0, 0);">brk</font>/<font style="color:rgb(0, 0, 0);">mmap</font>），从用户态切换到内核态；
  3. 操作系统在内核态查找空闲内存块、更新内存管理数据结构，再切换回用户态返回内存地址。

而栈操作（局部变量分配 / 回收）仅需修改栈顶指针（如 <font style="color:rgba(0, 0, 0, 0.85) !important;">push</font>/<font style="color:rgba(0, 0, 0, 0.85) !important;">pop</font> 指令），全程在用户态完成，无需内核介入，因此速度远快于堆。

数据结构的角度：#

  • 堆分为最大堆和最小堆，可以实现堆排序（引出堆排序以及优先级队列）
  • 栈的特点是先进后出原则

16、构造函数可以成为虚函数吗？#

不可以，因为调用虚函数需要通过虚函数表指针找到虚函数表，而系函数表指针是在构造函数中被赋值的，在执行构造函数之前无法通过虚函数表指针找到虚函数表，也就不能调用虚函数，如果构造函数称为虚函数的话将无法被调用。

17、析构函数可以是虚函数吗？#

可以，一般在继承中如果这个类会产生多态那么析构函数一定要设置为虚函数，否则在多态时，由于编译器会把父类和子类的构造函数同一命名，那么此时析构函数为函数隐藏，只会调用指针类型里面的析构函数发生了内存泄漏，如果基类中析构函数为虚函数，那么将会在子类的析构函数中去调用父类的析构函数，以此来避免内存泄漏的产生。

18、什么是内存泄漏？#

内存泄漏memory leak，是指程序在申请内存后，无法释放已申请的内存空间，一次内存泄漏危害可以忽略，但是内存泄漏堆积后果很严重，无论多少内存，迟早会被占用光。

内存泄漏是指你向系统申请分配内存进行使用（new），可是使用完了以后却不归还（delete），结果你申请到的那块内存你自己也不能访问（也许内存丢了），而系统也不能在次将它分配。了解下如何查找解决内存泄漏vs提供了工具

void func() {
    int* p = new int[10]; // 申请堆内存
    // 使用p后未执行 delete[] p;  → 内存泄漏
}

void func() {
    int* p = new int;
    p = new int[5]; // 原new int的内存地址被覆盖，无法释放
    delete[] p; // 仅释放了new int[5]的内存，原内存泄漏
}

void func() {
    int* p = new int;
    if (someError) {
        throw std::exception(); // 抛出异常，直接跳出函数
    }
    delete p; // 异常发生时，此句无法执行 → 内存泄漏
}

class A {
private:
    int* data;
public:
    A() { data = new int[100]; }
    // 未定义析构函数 ~A() { delete[] data; } → 对象销毁时data指向的内存泄漏
};

25、既然有了malloc和free为什么还要有new和delete？#

因为C语言没有对象，而C++是面向对象语言，C++中类或结构体创建的对象在释放时需要调用析构函数去释放对象内部的指针指向的堆区内存，在创建对象时需要调用构造函数给成员变量赋值。而malloc不会调用构造函数，free不会调用析构函数。new会先调用malloc申请内存，然后在调用构造函数赋值（初始化时初始化参数列表），delete会先调用析构函数释放对象内的成员变量指向的堆区内存，在调用free。（这里要注意为啥不是先调用free再调用析构。）delete在释放数组时要主要加中括号。

被delete或free释放的内存会直接返还给操作系统吗

被 delete 或 free 释放的内存通常不会直接返还给操作系统，而是由底层的内存分配器（如 Linux 下的 ptmalloc、Windows 下的 HeapAlloc、C++17 后的 std::pmr 等）管理，放入其维护的内存池（Memory Pool） 中，供后续内存申请复用。这一机制的核心目标是平衡性能、内存碎片和操作系统交互成本，并非单纯解决内存碎片问题。

一、为什么不直接返还给操作系统？

直接将释放的内存归还给操作系统（即“退还给内核”）存在显著弊端，这是内存分配器选择“内存池缓存”的根本原因：

1. 操作系统交互成本极高

应用程序与操作系统内核的内存交互（如通过 brk/sbrk 调整进程堆大小，或 mmap 申请匿名内存）属于系统调用，需要从“用户态”切换到“内核态”，这个过程涉及上下文保存、权限校验等操作，耗时远高于用户态的内存操作（通常差几个数量级）。
如果每次 free 都触发系统调用归还内存，后续 malloc 又需重新调用系统调用申请，会严重拖慢程序性能。

2. 内存“粒度不匹配”

操作系统管理的内存单位是页（Page）（通常 4KB 或 8KB），而应用程序申请的内存往往是小粒度的（如 malloc(10) 申请 10 字节）。

  • 若释放 10 字节就试图归还内核，内核无法回收“部分页面”，只能回收整个页面——但该页面中可能还存在其他被应用程序使用的内存（即“页面内有活跃数据”），导致无法回收。
  • 因此，只有当一整个页面的内存都被释放（且无活跃数据）时，内存分配器才可能将其归还给操作系统（称为“内存紧缩”或“页面卸载”）。

3. 复用内存更高效

内存池缓存释放的内存后，后续 malloc 可以直接在用户态的内存池中查找合适的空闲块，无需与内核交互，极大提升了分配速度。这对频繁申请/释放小内存的场景（如链表、字符串操作）至关重要。

二、以 ptmalloc（glibc 标准内存分配器）为例：核心实现逻辑

ptmalloc 是 Linux 下最经典的内存分配器（C 语言 malloc/free、C++ new/delete 底层默认依赖它），其设计完美体现了“内存池缓存”思想，核心机制包括堆区管理和空闲块复用：

1. 内存分配的两个“源头”

ptmalloc 向内核申请内存有两种方式，对应不同粒度的内存需求：

  • brk/sbrk 系统调用：用于申请“小内存”（通常 ≤ 128KB），通过调整进程“堆顶指针”（program break）扩展/收缩堆空间。这部分内存被称为“主分配区”（main arena）的内存池。
  • mmap 系统调用：用于申请“大内存”（通常 > 128KB），直接向内核映射一块独立的匿名内存区域（不属于堆，属于“内存映射区”）。这部分内存释放后会直接归还给内核（因为粒度是页面级，且独立存在，无页面内活跃数据问题）。

2. 小内存的“缓存与复用”（核心）

对于 brk 申请的小内存，free 后不会归还给内核，而是被 ptmalloc 标记为“空闲块”，放入对应的空闲块链表（按大小分类，如“16字节链表”“32字节链表”“64字节链表”等）：

  • 分配时：malloc 根据申请大小，直接在对应大小的空闲块链表中查找合适的块（“最佳适配”或“首次适配”策略），若找到则直接复用；若找不到，再向内核扩展堆空间。
  • 释放时：free 会先检查释放的块是否能与相邻的空闲块“合并”（解决“外部碎片”问题），合并后再插入对应的空闲块链表，等待复用。

只有当整个堆区的空闲内存足够多（且长时间未被复用）时，ptmalloc 才会通过 sbrk 收缩堆顶，将部分空闲内存归还给内核——但这种情况较少见，大部分小内存会一直缓存于内存池中。

3. 如何解决内存碎片？

内存分配器的“内存池缓存”并非直接解决碎片，而是通过空闲块合并和分类管理缓解碎片：

  • 外部碎片：释放的块与相邻空闲块合并，避免小块空闲内存分散在内存中无法被大申请使用。
  • 内部碎片：按固定大小划分空闲块（如申请 10 字节时，分配 16 字节对齐的块），虽有少量内部浪费，但避免了碎片过度分散。

但内存池缓存本身可能导致“假阳性内存占用”——即应用程序实际使用的内存很少，但内存池缓存了大量空闲内存，导致进程的“常驻内存（RSS）”居高不下（这也是为什么 free 后 top 命令看到的内存占用没减少的原因）。

三、特殊情况：内存会直接返还给操作系统吗？

存在两种例外场景，释放的内存会直接归还给内核：

  1. 大内存释放（mmap 分配的内存）：如前所述，ptmalloc 对 > 128KB 的内存用 mmap 分配，free 时会直接调用 munmap 归还给内核，因为这类内存独立存在，无页面内活跃数据问题，且避免缓存大内存造成浪费。
  2. 内存分配器主动“紧缩”：当系统内存紧张，或内存池中的空闲内存长期未被复用时，部分内存分配器（如 ptmalloc、tcmalloc）会主动将连续的大块空闲内存归还给内核，以释放系统资源。

面试延伸：如何验证“内存未直接返还”？#

可以通过 Linux 命令行工具验证：

  1. 写一个程序：malloc(100) 申请内存，free() 释放后进入死循环。
  2. 用 top 或 ps aux 查看进程的 RSS（常驻内存）——会发现 free 后 RSS 并未减少，因为释放的 100 字节仍在内存池中缓存。
  3. 若将 malloc(100) 改为 malloc(200*1024)（200KB，大内存），free() 后 RSS 会明显下降，因为内存被 munmap 归还给了内核。


这一机制是内存分配器的核心优化点，也是大厂面试中考察“底层内存管理”的高频考点——它不仅关乎“是否返还内存”，更涉及操作系统交互、性能优化、碎片处理等深层原理。

29、常函数内能否修改静态成员？#

可以，常函数内不能修改非静态成员，因为访问非静态成员需要使用this指针，例如：在类A中的一个常函数内this指针类型为：const A* const this;*this被const修饰表示不能通过this指针修改对象，我们在访问非静态成员时需要使用this指针，所以不能被修改。访问静态成员时不需要使用this指针区分是哪个对象的成员，因为静态成员是共享的单独存放在静态区不占对象的内存，所以常函数内可以修改静态成员。

31、C++生成可执行文件的四个步骤#

  • 预处理：预处理器根据以字符#开头的命令，修改原来的C++程序。这个阶段主要处理#include、#define等预处理指令，将头文件插入到程序中，并进行宏替换等操作。
  • 编译：编译器将预处理后的文件作为输入，对其进行语法分析和予以分析等处理。这些处理会将代码转换为中间代码，中间代码是汇编语言。
  • 汇编：汇编器将编译阶段生成的中间代码翻译成为机器语言指令（01二进制），并将结果保存到目标文件中。
  • 链接：链接器将各个目标文件以及程序所需要的库文件进行链接，生成可执行文件。链接器会解决目标文件之间的引用关系，并将他们组合成一个完整的程序。

32、什么是悬挂指针#

悬挂指针，也叫野指针，是指向非法内存地址的指针，即无法正常使用的指针。

33、什么时候会出现悬挂指针#

  • 使用未初始化的指针：在定义指针变量后没有对其进行初始化，这是出现悬挂指针最典型的情形。
  • 指针所指的对象已经消亡：当指针指向的对象的生命周期结束后，对象已经消亡，但仍然使用该指针访问该对象，就会出现运行时错误。
  • 指针释放后为置空：指针被free或者delete释放后，没有置为NULL，此时指针指向的是“垃圾”内存。
  • 在C语言中，realloc函数使用不当：当使用 realloc 扩展内存时，若原内存块后面没有足够空间，realloc 会在堆区重新分配一块满足大小的新内存，并将原内存块中的数据复制到新内存中。此时原内存块会被自动释放，原指针将指向已被释放的内存（成为悬挂指针）。

34、#define和const有什么区别#

  • define定义的常量没有类型，const定义的常量带有类型
  • 编译器对其处理不同，define定义的宏在预处理阶段被替换可能有多个拷贝，const定义的变量在编译时确定值，只有一个拷贝（为什么会有一个拷贝，参考const修饰的变量能被修改cast_const<>()<font style="color:rgb(0, 0, 0);">const</font>常量在编译阶段确定值，编译器会为其分配固定内存地址，所有对该常量的引用都会指向同一地址；即使在多个文件中声明（需用<font style="color:rgb(0, 0, 0);">extern const</font>），链接时也会被解析为同一个内存地址，避免重复存储。）

36、讲一下C++的定义和声明#

  • 声明：主要是向编译器告知某个变量、函数或类型的存在，但不分配实际的存储空间或详细定义其行为。声明只是告诉编译器“有这么个东西”。
  • 定义：不仅声明了变量、函数或类型的存在，还会为其分配存储空间（对于变量）或详细描述其行为（对于函数等）。一个变量或函数在程序中只能被定义一次。

37、C++将引用作为返回值的好处和应该遵守的规则#

  • 减少内存开销：在内存中不产生返回值的副本，避免了不必要的内存分配和复制操作。
  • 提高效率：可以直接返回对象本身，而不需要创建临时对象，从而提高程序的运行效率。
  • 支持链式操作：返回引用可以执行链式操作，使得代码更加简洁和易读。
  • 流操作符重载返回值应声明为引用：为了保证连续使用流操作符（«和»）重载返回值时，操作的时同一个对象。
  • +-*/四则运算符不能返回引用：这是因为四则用算符通常用于产生新的值，这里相当于返回局部变量里的引用，而不是直接修改操作数。
  • 不能返回局部变量的引用：因为局部变量在函数返回后会被销毁，返回其引用会导致引用指向无效的内存区域，可能引发运行时错误。
  • 可以返回类成员的引用，但最好是const类型：这样可以避免意外修改类成员的值，同时也能提高代码的可读性和可维护性。

38、成员函数是通过什么来区分不同对象的成员变量的？#

成员函数是通过对象的指针（this指针）来区分不同对象的成员变量的。

在类的成员函数内部，有一个隐含的指针this，它指向调用该成员函数的对象。当不同的对象调用同一个成员函数时，this指针就会指向各自对应的对象，从而能够准确地访问和操作该对象的成员变量，将不同对象的数据区分开来。

例如，有一个类Person，包含成员变量age，当不同的Person对象调用成员函数来修改age时，this指针会明确指向具体的那个对象，使得修改的是当前对象的age，而不是其他对象的。

39、C++编译器为类提供的四个缺省函数是什么#

  • 默认构造函数：在没有显示定义构造函数时，编译器会生成一个无参数、空实现的默认构造函数。
  • 拷贝构造函数：浅拷贝，成员变量的赋值操作，不会产生新的堆区内存。
  • 赋值运算符重载函数：会有浅拷贝的问题，因为也是成员变量的赋值操作。
  • 析构函数：在对象销毁时执行一些清理工作。

补充：C++11 后的扩展

默认移动构造函数：当类没有显式定义移动构造函数，且未定义拷贝构造、拷贝赋值、析构函数时，编译器会生成，用于 “窃取” 临时对象的资源（避免深拷贝，提升效率）。

默认移动赋值运算符重载：类似移动构造函数，用于 “用临时对象给已有对象赋值”，同样基于资源窃取。

40、在类外有什么办法可以访问非公有成员#

  • 通过友元：可以将其他函数或类声明为该类的友元，友元就可以访问该类的非公有成员。
  • 通过类的成员函数间接访问：类自身提供一些公有成员函数来操作非公有成员，在类外通过调用这些公有成员函数来达到访问非公有成员的目的。

41、在继承关系中，在栈区创建子类对象构造和析构的调用顺序是什么？#

  • 构造函数：先调用父类的构造函数，再调用子类的构造函数
  • 析构函数：先调用子类的析构函数，再调用父类的析构函数

42、C++空类大小为多大？#

1个字节。首先理解什么是对象：凡是有类型的内存都是对象，也就是说明对象一定是占有内存的，并且一个对象占有一份独一无二的内存，为了满足这一要求，编译器会为空类分配1字节的内存空间，使得每个空类对象都有唯一的地址。需要注意的是，当空类作为基类时，其大小可能会被优化为0字节。这是因为在这种情况下，空类的唯一作用时作为基类，而不需要占用实际的内存空间。这种优化被称为“空白基类最优化”。

43、内联函数可以是虚函数吗？#

在C++中，虚函数可以是内联函数，但是要注意一些限制。内联函数是在编译时将函数体直接插入到调用处，以减少函数调用的开销，提高程序的执行效率。而虚函数时通过动态绑定来实现多态性的，即在运行时根据对象的实际类型来确定调用哪个函数。

当通过类的对象来调用虚函数时，编译器在编译时可以确定对象的类型，因此可以将虚函数内联展开。但当通过类的指针或引用调用虚函数时，由于需要在运行时确定对象的实际类型，所以编译器无法进行内联优化。

另外，即使虚函数被声明为内联函数，编译器也可能根据具体情况决定是否真正进行内联。例如，如果函数体过大、存在循环或递归等复杂结构，编译器可能会选择不进行内联，而是采用普通的函数调用方式。

44、抽象类为什么不能创建对象#

抽象类通常包含至少一个纯虚函数。纯虚函数没有具体的实现，它知识一个声明，用于定义子类必须实现的接口。

意义：

  • 强制继承：它确保了派生类必须提供这些关键函数的具体实现，以满足特定的行为要求。
  • 定义接口规范：抽象类定义了一套接口标准，让派生类遵循。

接口的理解：

接口可以指一组相关函数的集合，这些函数定义了一个类或一组类所具有的行为。

常见的实现接口的方式是通过抽象类，其中包含纯虚函数，这些纯虚函数构成了类所需要实现的接口部分。比如一个Shape抽象类可能定义了draw（）、area（）等纯虚函数作为接口，具体的形状类（如圆形、三角型等）需要继承并实现这些接口。有时候也会将一组仅包含纯虚函数、没有数据成员的抽象类专门称为接口类。

45、在成员函数中可以delete this吗？delete之后对象还可以使用吗？#

如果想在成员函数中调用delete，需要保证该成员函数是该对象最后一个调用的函数，并且在之后不访问该对象的任何成员。因为delete之后该对象就会被销毁。所以delete之后该对象不能使用，在多线程下delete还会引发其他并发问题。

46、C++如何阻止类创建对象#

  • 定义为抽象类：将类定义为抽象类可以阻止直接实例化改类。
  • 将构造函数声明为私有：将类的构造函数声明为私有可以阻止在类外实例化对象。但是类内静态的成员函数仍然可以调用构造函数创建对象，因为公有的静态成员函数可以直接通过类名调用：
#include<iostream>

	int num;
	A(int num) {
		cout << "A构造" << endl;
		this->num = num;
		cout << this->num << endl;
	}
public:
	static void fun(int n) {
		A a(n);
	}
};

int main() {
	A::fun(1);
	return 0;
}

48、C++的内存分配#

  • 堆区：一般由程序员自动分配，如果程序员没有释放，也会在整个程序结束时被操作系统回收。
  • 栈区：由编译器自动分配和释放，存放运行函数分配的局部变量，函数参数，返回数据，返回地址等，其操作类似于数据结构中的栈。
  • 全局区：（静态区static）：存放全局变量，静态变量，常量。结束后由系统释放。
  • 常量区：（字符串常量区）：存放常量字符串，程序结束后由系统释放。
  • 代码区：存放函数体（类成员函数和全局区）的二禁止代码。

49、声明和定义的区别#

声明是告诉编译器名字的存在，定义是为名字分配内存并实现其功能。在使用变量或函数之前，必须先进行声明或定义。

50、x=x+1，x+=1，x++哪个效率高？#

由于x++需要创建一个临时变量来保存x的旧值，所以他比其他两个效率稍慢，但是由于CPU处理数据的速度太快了，所以可以忽略不计。

  1. 效率理论排序（无编译器优化时）：<font style="color:rgba(0, 0, 0, 0.85) !important;">x += 1</font> ≈ <font style="color:rgba(0, 0, 0, 0.85) !important;">++x</font> > <font style="color:rgba(0, 0, 0, 0.85) !important;">x = x + 1</font> > <font style="color:rgba(0, 0, 0, 0.85) !important;">x++</font>（差距主要体现在非基本类型的临时对象开销上）
  2. 实际开发中的选择：
    • 对于基本类型：三者效率无差异，可根据语义选择（如 <font style="color:rgb(0, 0, 0);">x++</font> 用于需要旧值的场景）。
    • 对于自定义类型：优先使用 <font style="color:rgb(0, 0, 0);">x += 1</font> 或 <font style="color:rgb(0, 0, 0);">++x</font>，避免 <font style="color:rgb(0, 0, 0);">x = x + 1</font> 和 <font style="color:rgb(0, 0, 0);">x++</font> 的临时对象开销。
    • 可读性优先：过度纠结基本类型的效率差异意义不大，代码的清晰性更重要。

简言之，<font style="color:rgba(0, 0, 0, 0.85) !important;">x++</font> 因必须保存旧值而理论上效率最低，但在编译器优化下，基本类型的差异可忽略；非基本类型则需优先选择 <font style="color:rgba(0, 0, 0, 0.85) !important;">+=</font> 或前置 <font style="color:rgba(0, 0, 0, 0.85) !important;">++</font> 以减少开销。

51、（重点）全局变量可不可以定义在被多个源文件包含的头文件中？为什么？#

  • 可以，但是要声明为static。
  • 可以在不同的C文件中声明同名的全局变量，前提是其中只能有一个C文件对此变量赋初值，此时链接不会报错。

如果在头文件中定义全局变量，当该头文件被多个文件include时，该头文件中的全局变量就会多次被定义，导致重定义，因此不能在头文件中定义全局变量。所以在头文件中定义全局变量需要定义为静态全局变量，这样当头文件被引用时就不会导致重定义。

  • 绝对禁止在头文件中定义普通全局变量：会导致重定义错误。
  • 不推荐在头文件中定义静态全局变量：会产生多份独立副本，导致数据不一致和内存浪费。
  • 正确方式：头文件用<font style="color:rgb(0, 0, 0);">extern</font>声明，源文件中唯一定义，确保全局变量在程序中只有一份内存，且可跨文件访问。

53、C++构造函数有几种，分别有什么作用？#

  • 默认构造函数：它可以在没有提供任何初始化参数的情况下被调用，用于对类的数据成员进行默认初始化。
  • 一般构造函数（带参数的构造函数）：通过接收不同的参数，实现对类的数据成员进行灵活的初始化，以满足各种具体场景的需求。
  • 拷贝构造函数：用于创建一个与现有对象完全相同的新对象，例如在对象的复制操作中发挥作用。
  • 移动构造函数：用于处理右值引用，提高资源移动的效率，特别是在涉及临时对象等情况时。

54、C++类对象的初始化顺序#

  • 基类初始化：如果类有基类，那么基类的构造函数会首先被调用，按照基类在派生列表中的顺序进行
  • 成员类对象初始化：如果类中包含成员类对象，那么成员类对象的构造函数会在基类构造函数之后被调用，按照成员类对象在类中声明的顺序进行。
  • 自身构造函数：最后，类自身的构造函数会被调用，完成对象的初始化。

55、sizeof和strlen的区别#

56、对拷贝构造函数和赋值运算符的认识#

默认生成的拷贝构造和赋值运算符都会产生同一块内存被释放两次的问题，在用法上拷贝构造是创建新的对象，而赋值运算符是给已经存在的对象赋值，在实现上因为赋值运算符是给已经存在的对象重新赋值所以需要检查源对象是否指向堆区内存，如果指向需要释放。

#include<iostream>
using namespace std;

class A {
public:
	int* p;
	A() : p(nullptr){}
	A(int n) {
		p = new int(n);
	}

57、vector中的push_back()和emplace_back()的区别，以及使用场景#

  • 参数传递方式：push_back()接受一个元素对象作为参数，并将其复制或移动到向量的末尾。而emplace_back()接收构造函数的参数，直接在向量的尾部构造元素，避免了不必要的复制或移动操作。

58、解释一下map和unordered_map的区别以及使用场景#

  • 数据结构：map基于红黑树实现，元素是有序存储的；而unordered_map基于哈希表实现，元素存储时无序的。
  • 查找效率：一般情况下，unordered_map的平均查找效率更高，接近常数时间复杂度，但在最坏情况下可能会退化；map的查找性能相对稳定为log n。
  • 元素顺序：map中的元素按照特定的顺序（通常是键的顺序）排列；unordered_map没有固定的元素顺序。
  • 使用场景：
    • 当需要按照键的顺序对元素进行有序访问时，比如需要顺序遍历，优先选择map。
    • 当对元素的插入、查找效率要求较高，不太关心元素的顺序时，适合使用unordered_map，比如快速查询大量的数据。
    • 如果数据量较大，且对性能要求很敏感，且不需要有序性，通常使用unordered_map表现更好。例如，在一个需要根据ID快速查找用户信息的场景，如果对用户信息的显示顺序没有严格要求，可能会选择unordered_map；而如果需要按照用户ID从小到大显示用户列表，那么map更合适。

  • 定义抽象基类：当希望创建一个基类，它表示一组相关类的通用接口，但本身不能被实例化时，通过定义纯虚函数来明确该接口。

63、vector是怎么进行扩容的#

数组的起始容量为0，GCC 是2倍进行扩容，即如果原来容器的容量为5扩容后容量大小为 10，VS 中扩容是 1.5 倍。扩容需要新申请一块内存空间然后把旧内存的数据拷贝过去，并日释放日内存空间，不仅仅push back()容量不够时会扩容，resize时也会扩容。resize时传入的参数不仅仅大于 size 的大小如果也大于容量大小时就会进行扩容。

	cout << "size=" << vec.size() << " " << "capacity=" << vec.capacity() << endl;
	vec.resize(50);//当resize的参数大于原容量的1.5倍时按照参数大小进行扩容

64、什么是迭代器失效?#

迭代器的实现是基于指针的，在使用 vector 等容器时，迭代器失效是指在某些操作之后，原本有效的迭代器可能变得不再有效或产生不确定的行为。

当在 vector 中进行插入或删除元素操作时，如果导致了容器的重新分配内存(如扩容)，那么指向原内存位置的选代器就失效了。

例如，当不断向一个 vector 中插入元素，导致多次扩容后，之前获取的选代器就不能再继续使用了，因为元素的存储位置可能已经发生了改变。如果继续使用失效的迭代器进行操作，可能会导致程序出现未定义的行为甚至崩溃。

//定义一个队列节点的结构体
struct QueueNode {
	int data;
	QueueNode* next;
	QueueNode(int val):data(val),next(nullptr){}
};
class Queue {
private:
	QueueNode* front;//队列头指针
	QueueNode* rear;//队列尾指针
public:
	Queue():front(nullptr),rear(nullptr){}
	//检查队列是否为空
    bool isEmpty() {
		return front == nullptr;
	}
	//入队操作
	void enqueue(int val) { 
		QueueNode* newNode = new QueueNode(val);
		if (isEmpty()) {
			front = rear = newNode;
		}
		else {
			rear->next = newNode;
            rear = newNode;
		}
	}
	//出队操作
	int dequeue() {
		if (isEmpty()) {
            cout << "队列为空" << endl;
            return -1;
		}
		int val = front->data;
        QueueNode* temp = front;
        front = front->next;
		if (front == nullptr) {
            rear = nullptr;
		}
        delete temp;
        return val;
	}
	//打印队列
    void printQueue() {
		QueueNode* cur = front;
		while (cur) {
			cout << cur->data << " ";
			cur = cur->next;
		}
		cout << endl;
	}
};

    Queue q;
    q.enqueue(1);
    q.enqueue(2);
    q.enqueue(3);
	q.printQueue();
    cout << q.dequeue() << endl;
	q.printQueue();

	return 0;
}

class Node {
public:
    int data;
    Node *next;
    Node(int data) {
        this->data = data;
        this->next = nullptr;
    }
};
class Stack {
private:
    Node* top;
public:
    Stack(){top = nullptr;}
    //入栈
    void push(int element) {
        Node* newNode  =new Node(element);
        newNode->next = top;
        top = newNode;
    }
    //获取栈顶元素
    int gettop() {
        if (top == nullptr) {
            cout<< "栈为空" << endl;
            return -1;
        }
        return top->data;
    }
    //弹出栈顶元素
    int pop() {
        if (top == nullptr) {
            cout<< "栈为空" << endl;
            return -1;
        }
        int element = top->data;
        Node* temp = top;
        top = top->next;
        delete temp;
        return element;
    }
    //判断栈是否为空
    bool isEmpty() {
        return top == nullptr;
    }
};

int main()
{
    Stack s;
    s.push(1);
    s.push(2);
    s.push(3);
    cout<< s.gettop() << endl;
