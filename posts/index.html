面试常见问题整理

1、面向对象 什么是面向对象？ 面向对象： 面向对象（Object-Oriented Programming，简称 OOP）是一种编程思想，核心是将现实世界中的具体事物抽象为对象，对象具有属性（数据）和行为（操作数据的方法），同类对象的抽象为类。 面向对象三大特性： 封装： 将事物的属性和行为包装在一起成类，通过设置访问权限（public、private，protected）来控制对外暴露。 继承： 通过现有的类来实现新的类，继承的类（子类）拥有被继承类（父类）的属性和行为，可以在原有功能的基础上进行扩展。提高了代码的复用性。是多态的前提。 多态： 是指同一接口在不同对象上表现出不同的行为。当父类的引用 / 指针指向子类的对象时，调用同一方法会执行子类的实现（而非父类），实现 “调用方式统一，执行结果不同”。 多态的分类：从“行为差异的时机”划分 多态的核心是“同一接口，不同行为”，但“行为差异何时确定”决定了其分类，这与面向对象中“类的抽象”和“对象的具体”紧密相关： 1. 静态多态（编译期多态）：基于“类的抽象”在编译时确定行为 静态多态的行为差异在编译阶段就已确定，依赖于“类的静态接口定义”，不涉及对象的动态类型。它的核心是“同一函数名，根据输入参数的差异（类型、个数等）匹配不同实现”，本质是编译器对“类或函数的抽象定义”进行静态解析。 实现方式：函数重载、运算符重载。 这是面向对象中“封装”特性的体现：类通过封装不同参数的同名方法，对外提供统一接口（函数名），内部根据参数差异处理不同逻辑。 示例（类内函数重载，静态多态）： // 封装了“计算面积”行为的 Shape 类 class Shape { public: // 重载1：计算圆形面积（参数：半径） double area(double radius) { return 3.14 * radius * radius; } // 重载2：计算矩形面积（参数：长、宽） double area(double length, double width) { return length * width; } }; int main() { Shape shape; cout << shape.area(2.0) << endl; // 编译时匹配“圆形面积”，输出 12.56 cout << shape.area(2.0, 3.0) << endl; // 编译时匹配“矩形面积”，输出 6.0 return 0; } 这里的“多态”是Shape类通过封装不同参数的area方法实现的，编译器在编译时就根据参数确定了调用哪个版本，与对象的动态类型无关（静态多态不依赖继承）。 2. 动态多态（运行期多态）：基于“对象的具体”在运行时确定行为 动态多态的行为差异在程序运行阶段才确定，依赖于“对象的实际类型”，是面向对象三大特性协同作用的结果： 依赖继承：必须存在父类与子类的层级关系（子类是父类的具体实现）； 依赖封装：父类通过virtual关键字封装虚函数（统一接口），子类重写该函数（具体实现）； 依赖“对象的抽象与具体”：通过父类的引用/指针指向子类对象（抽象接口指向具体实例），调用接口时动态匹配对象的具体实现。 这完全符合面向对象“将现实事物抽象为对象，同类对象抽象为类”的思想——比如“动物（父类）”是抽象类，“猫、狗（子类）”是具体对象，“叫（虚函数）”是统一接口，不同动物的叫声（子类重写）是具体行为。 动态多态的实现：虚函数+继承+父类引用/指针 动态多态的实现严格依赖三大条件（与面向对象特性深度绑定）： 继承关系（继承特性）：子类必须公有继承父类，保证父类接口能被子类复用和扩展； 虚函数重写（封装+继承）：父类用virtual声明虚函数（封装统一接口），子类用override重写该函数（扩展具体实现），且函数签名（名称、参数、返回值）必须完全一致； 父类引用/指针（抽象与具体的关联）：通过父类的引用或指针指向子类对象，调用虚函数时，程序会根据对象的实际类型（而非指针/引用的声明类型）执行对应实现。 底层机制：虚函数表（vtable）与虚指针（vptr） 每个包含虚函数的类（父类/子类）会有一个虚函数表（vtable）（类级别的全局表），存储该类所有虚函数的地址。子类重写父类虚函数后，会用子类函数地址替换vtable中对应位置的父类函数地址； 每个对象会隐含一个虚指针（vptr）（对象级别的隐藏成员），指向所属类的vtable。当通过父类指针/引用调用虚函数时，程序通过vptr找到vtable，再定位到具体函数地址（运行时动态绑定）。 示例（动态多态，结合继承与封装）： // 抽象父类：动物（封装虚函数接口） class Animal { public: virtual void bark() = 0; // 纯虚函数（仅声明接口，无实现） }; // 子类：狗（继承并扩展父类接口） class Dog : public Animal { public: void bark() override { // 重写虚函数，具体实现 cout << "汪汪" << endl; } }; // 子类：猫（继承并扩展父类接口） class Cat : public Animal { public: void bark() override { // 重写虚函数，具体实现 cout << "喵喵" << endl; 1. 两数之和 要解决“两数之和”问题，使用哈希表可以高效地在O(n)时间复杂度内完成。以下是具体实现： 解题思路： 哈希表作用：使用unordered_map存储已经遍历过的数值及其对应的索引，实现O(1)时间复杂度的查找。 遍历数组：对每个元素nums[i]，计算与目标值的差值（补数）complement = target - nums[i]。 查找补数：如果补数已存在于哈希表中，说明之前遍历过的元素中存在与当前元素之和为目标值的数，直接返回两个索引。 存储当前元素：如果补数不存在，则将当前元素及其索引存入哈希表，继续遍历。 #include <vector> #include <unordered_map> using namespace std; class Solution { public: vector<int> twoSum(vector<int>& nums, int target) { unordered_map<int, int> numMap; // 存储数值到索引的映射 for (int i = 0; i < nums.size(); ++i) { int complement = target - nums[i]; // 计算需要找到的补数 // 检查补数是否已存在于哈希表中 if (numMap.find(complement) != numMap.end()) { // 找到则返回两个数的索引 return {numMap[complement], i}; } // 未找到则将当前数值和索引存入哈希表 numMap[nums[i]] = i; } // 题目保证有解，这里只是为了语法完整 return {}; } }; 复杂度分析： 时间复杂度：O(n)，其中n是数组长度，每个元素最多被遍历一次，哈希表的插入和查找操作平均为O(1)。 空间复杂度：O(n)，最坏情况下需要存储所有元素到哈希表中。 这种方法通过空间换时间，高效地解决了两数之和问题，避免了暴力解法的O(n²)时间复杂度。 49. 字母异位词分组 要解决“字母异位词分组”问题，我们可以利用哈希表结合字符排序的方法，高效地将异位词（字母相同但顺序不同的单词）分组。以下是具体实现： 解题思路 异位词的核心特征是：对单词中的字符进行排序后，得到的字符串完全相同。例如，“listen” 和 “silent” 排序后都是 “eilnst”。因此，我们可以： 以“排序后的字符串”作为哈希表的键（key），确保所有异位词对应同一个键。 以“异位词组成的列表”作为哈希表的值（value），将所有排序后相同的单词存入同一列表。 遍历所有单词，完成分组后，提取哈希表中的所有值，即为最终结果。 #include <vector> #include <string> #include <unordered_map> #include <algorithm> using namespace std; class Solution { public: vector<vector<string>> groupAnagrams(vector<string>& strs) { // 哈希表：key为排序后的字符串，value为异位词列表 unordered_map<string, vector<string>> groups; for (const string& s : strs) { // 对当前字符串排序，得到key string key = s; sort(key.begin(), key.end()); // 将原字符串加入对应的分组 groups[key].push_back(s); } // 提取所有分组到结果中 vector<vector<string>> result; for (const auto& pair : groups) { result.push_back(pair.second); } return result; } }; 复杂度分析 时间复杂度：O(n × k log k)，其中 n 是字符串的数量，k 是字符串的最大长度。每个字符串需要排序（时间复杂度 O(k log k)），总共有 n 个字符串，因此整体为 O(n × k log k)。 空间复杂度：O(n × k)，哈希表需要存储所有字符串，总空间取决于所有字符串的总长度。 关键说明 排序是核心：通过排序将异位词转换为相同的键，确保分组正确性。 哈希表高效映射：利用 unordered_map 实现键到列表的快速映射，插入和查找操作的平均时间复杂度为 O(1)。 这种方法简洁且高效，是解决异位词分组问题的经典思路。 ...
