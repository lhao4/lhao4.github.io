面试常见问题整理

1、面向对象 什么是面向对象？ 面向对象： 面向对象（Object-Oriented Programming，简称 OOP）是一种编程思想，核心是将现实世界中的具体事物抽象为对象，对象具有属性（数据）和行为（操作数据的方法），同类对象的抽象为类。 面向对象三大特性： 封装： 将事物的属性和行为包装在一起成类，通过设置访问权限（public、private，protected）来控制对外暴露。 继承： 通过现有的类来实现新的类，继承的类（子类）拥有被继承类（父类）的属性和行为，可以在原有功能的基础上进行扩展。提高了代码的复用性。是多态的前提。 多态： 是指同一接口在不同对象上表现出不同的行为。当父类的引用 / 指针指向子类的对象时，调用同一方法会执行子类的实现（而非父类），实现 “调用方式统一，执行结果不同”。 多态的分类：从“行为差异的时机”划分 多态的核心是“同一接口，不同行为”，但“行为差异何时确定”决定了其分类，这与面向对象中“类的抽象”和“对象的具体”紧密相关： 1. 静态多态（编译期多态）：基于“类的抽象”在编译时确定行为 静态多态的行为差异在编译阶段就已确定，依赖于“类的静态接口定义”，不涉及对象的动态类型。它的核心是“同一函数名，根据输入参数的差异（类型、个数等）匹配不同实现”，本质是编译器对“类或函数的抽象定义”进行静态解析。 实现方式：函数重载、运算符重载。 这是面向对象中“封装”特性的体现：类通过封装不同参数的同名方法，对外提供统一接口（函数名），内部根据参数差异处理不同逻辑。 示例（类内函数重载，静态多态）： // 封装了“计算面积”行为的 Shape 类 class Shape { public: // 重载1：计算圆形面积（参数：半径） double area(double radius) { return 3.14 * radius * radius; } // 重载2：计算矩形面积（参数：长、宽） double area(double length, double width) { return length * width; } }; int main() { Shape shape; cout << shape.area(2.0) << endl; // 编译时匹配“圆形面积”，输出 12.56 cout << shape.area(2.0, 3.0) << endl; // 编译时匹配“矩形面积”，输出 6.0 return 0; } 这里的“多态”是Shape类通过封装不同参数的area方法实现的，编译器在编译时就根据参数确定了调用哪个版本，与对象的动态类型无关（静态多态不依赖继承）。 2. 动态多态（运行期多态）：基于“对象的具体”在运行时确定行为 动态多态的行为差异在程序运行阶段才确定，依赖于“对象的实际类型”，是面向对象三大特性协同作用的结果： 依赖继承：必须存在父类与子类的层级关系（子类是父类的具体实现）； 依赖封装：父类通过virtual关键字封装虚函数（统一接口），子类重写该函数（具体实现）； 依赖“对象的抽象与具体”：通过父类的引用/指针指向子类对象（抽象接口指向具体实例），调用接口时动态匹配对象的具体实现。 这完全符合面向对象“将现实事物抽象为对象，同类对象抽象为类”的思想——比如“动物（父类）”是抽象类，“猫、狗（子类）”是具体对象，“叫（虚函数）”是统一接口，不同动物的叫声（子类重写）是具体行为。 动态多态的实现：虚函数+继承+父类引用/指针 动态多态的实现严格依赖三大条件（与面向对象特性深度绑定）： 继承关系（继承特性）：子类必须公有继承父类，保证父类接口能被子类复用和扩展； 虚函数重写（封装+继承）：父类用virtual声明虚函数（封装统一接口），子类用override重写该函数（扩展具体实现），且函数签名（名称、参数、返回值）必须完全一致； 父类引用/指针（抽象与具体的关联）：通过父类的引用或指针指向子类对象，调用虚函数时，程序会根据对象的实际类型（而非指针/引用的声明类型）执行对应实现。 底层机制：虚函数表（vtable）与虚指针（vptr） 每个包含虚函数的类（父类/子类）会有一个虚函数表（vtable）（类级别的全局表），存储该类所有虚函数的地址。子类重写父类虚函数后，会用子类函数地址替换vtable中对应位置的父类函数地址； 每个对象会隐含一个虚指针（vptr）（对象级别的隐藏成员），指向所属类的vtable。当通过父类指针/引用调用虚函数时，程序通过vptr找到vtable，再定位到具体函数地址（运行时动态绑定）。 示例（动态多态，结合继承与封装）： // 抽象父类：动物（封装虚函数接口） class Animal { public: virtual void bark() = 0; // 纯虚函数（仅声明接口，无实现） }; // 子类：狗（继承并扩展父类接口） class Dog : public Animal { public: void bark() override { // 重写虚函数，具体实现 cout << "汪汪" << endl; } }; // 子类：猫（继承并扩展父类接口） class Cat : public Animal { public: void bark() override { // 重写虚函数，具体实现 cout << "喵喵" << endl; } }; // 统一接口：接收父类引用（抽象指向具体） void letAnimalBark(Animal& animal) { animal.bark(); // 运行时根据对象实际类型调用对应实现 } int main() { Dog dog; Cat cat; letAnimalBark(dog); // 传入Dog对象 → 输出“汪汪” letAnimalBark(cat); // 传入Cat对象 → 输出“喵喵” return 0;
