1. 两数之和 要解决“两数之和”问题，使用哈希表可以高效地在O(n)时间复杂度内完成。以下是具体实现： 解题思路： 哈希表作用：使用unordered_map存储已经遍历过的数值及其对应的索引，实现O(1)时间复杂度的查找。 遍历数组：对每个元素nums[i]，计算与目标值的差值（补数）complement = target - nums[i]。 查找补数：如果补数已存在于哈希表中，说明之前遍历过的元素中存在与当前元素之和为目标值的数，直接返回两个索引。 存储当前元素：如果补数不存在，则将当前元素及其索引存入哈希表，继续遍历。 #include <vector> #include <unordered_map> using namespace std; class Solution { public: vector<int> twoSum(vector<int>& nums, int target) { unordered_map<int, int> numMap; // 存储数值到索引的映射 for (int i = 0; i < nums.size(); ++i) { int complement = target - nums[i]; // 计算需要找到的补数 // 检查补数是否已存在于哈希表中 if (numMap.find(complement) != numMap.end()) { // 找到则返回两个数的索引 return {numMap[complement], i}; } // 未找到则将当前数值和索引存入哈希表 numMap[nums[i]] = i; } // 题目保证有解，这里只是为了语法完整 return {}; } }; 复杂度分析： 时间复杂度：O(n)，其中n是数组长度，每个元素最多被遍历一次，哈希表的插入和查找操作平均为O(1)。 空间复杂度：O(n)，最坏情况下需要存储所有元素到哈希表中。 这种方法通过空间换时间，高效地解决了两数之和问题，避免了暴力解法的O(n²)时间复杂度。 49. 字母异位词分组 要解决“字母异位词分组”问题，我们可以利用哈希表结合字符排序的方法，高效地将异位词（字母相同但顺序不同的单词）分组。以下是具体实现： 解题思路 异位词的核心特征是：对单词中的字符进行排序后，得到的字符串完全相同。例如，“listen” 和 “silent” 排序后都是 “eilnst”。因此，我们可以： 以“排序后的字符串”作为哈希表的键（key），确保所有异位词对应同一个键。 以“异位词组成的列表”作为哈希表的值（value），将所有排序后相同的单词存入同一列表。 遍历所有单词，完成分组后，提取哈希表中的所有值，即为最终结果。 #include <vector> #include <string> #include <unordered_map> #include <algorithm> using namespace std; class Solution { public: vector<vector<string>> groupAnagrams(vector<string>& strs) { // 哈希表：key为排序后的字符串，value为异位词列表 unordered_map<string, vector<string>> groups; for (const string& s : strs) { // 对当前字符串排序，得到key string key = s; sort(key.begin(), key.end()); // 将原字符串加入对应的分组 groups[key].push_back(s); } // 提取所有分组到结果中 vector<vector<string>> result; for (const auto& pair : groups) { result.push_back(pair.second); } return result; } }; 复杂度分析 时间复杂度：O(n × k log k)，其中 n 是字符串的数量，k 是字符串的最大长度。每个字符串需要排序（时间复杂度 O(k log k)），总共有 n 个字符串，因此整体为 O(n × k log k)。 空间复杂度：O(n × k)，哈希表需要存储所有字符串，总空间取决于所有字符串的总长度。 关键说明 排序是核心：通过排序将异位词转换为相同的键，确保分组正确性。 哈希表高效映射：利用 unordered_map 实现键到列表的快速映射，插入和查找操作的平均时间复杂度为 O(1)。 这种方法简洁且高效，是解决异位词分组问题的经典思路。 ...
